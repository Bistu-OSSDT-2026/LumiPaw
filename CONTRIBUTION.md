# LumiPaw 贡献说明

感谢你对 LumiPaw 项目的关注和贡献。

LumiPaw 欢迎开发者参与功能改进、问题修复和体验优化。

## 贡献流程

### 1. 提交 Issue

如果发现问题或有新的功能建议，请先创建 Issue。

Issue 内容建议包括：

- 问题描述
- 产生原因（如果已知）
- 期望解决方案
- 相关模块

---

### 2. 开发修改

在开始开发前，请确认对应 Issue，并根据项目模块划分进行修改。

项目主要模块：

- pet_system：桌宠显示和状态控制
- task_manager：任务管理
- economy：金币和奖励系统
- shop：装饰品商店
- ui：用户界面
- main：系统集成

---

### 3. 代码规范

提交代码时请遵守：

- 使用统一的变量命名方式；
- 保持模块独立，不直接修改其他模块内部数据；
- 新增功能需要提供对应测试；
- 避免提交无关文件。

---

### 4. Commit规范

提交信息应简洁明确。

示例：


Add task duration validation

Fix shop item classification issue

Update pet glow animation


---

### 5. Pull Request

提交 Pull Request 时需要说明：

- 修改内容；
- 解决的问题；
- 测试情况。

PR 标题应清晰描述修改内容。

示例：


Implement keyboard glow feedback system


---

## 开发原则

LumiPaw 采用模块化开发方式：

- 每个功能对应独立模块；
- 模块之间通过统一接口通信；
- 保持代码易维护和可扩展。

感谢所有参与 LumiPaw 开发和改进的贡献者。
