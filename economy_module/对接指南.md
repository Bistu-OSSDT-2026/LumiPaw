# 经济系统模块对接指南

## 给4号主程序的开发者

### 1. 如何导入经济系统模块

```python
# 在主程序中导入
from economy_module import (
    economy,
    event_listener,
    # 如果需要UI
    shop_ui
)
```

### 2. 监听任务完成事件（获取金币奖励）

```python
# 注册任务完成事件监听
def handle_task_completion(event):
    """处理任务完成事件，给予金币奖励"""
    if event["success"]:
        # 完成任务获得10金币
        economy.add_coins(10)
        print(f"任务完成奖励: +10金币")
        
# 注册监听
event_listener.register_task_callback(handle_task_completion)
```

### 3. 如何触发购买事件（如果用户购买物品）

```python
# 当用户购买物品时触发
def handle_purchase(item_id):
    """处理用户购买"""
    if economy.buy_item(item_id):
        # 触发购买事件
        event_listener.trigger_purchase_event(item_id, economy.get_shop_items()[item_id]["price"])
        print(f"购买 {economy.get_shop_items()[item_id]['name']} 成功！")
    else:
        print("金币不足，购买失败！")
```

### 4. 获取当前金币状态（用于UI显示）

```python
def get_coins_for_display():
    """获取当前金币数量用于显示"""
    return economy.get_coins()

def get_purchased_items_for_display():
    """获取已购买物品列表"""
    return economy.get_purchased_items()
```

### 5. 事件触发时机

#### 任务完成时（2号模块触发）
```python
# 2号模块完成任务后，4号需要触发
event_listener.trigger_task_completed(
    task_name="任务名称",
    duration=30,
    success=True  # 如果成功完成
)
```

#### 发光事件变化时（5号模块触发）
```python
# 5号模块检测到发光变化时触发
event_listener.trigger_glow_event(
    glow_type="keyboard",  # 或 "reward"
    value=2
)
```

### 8. 注意事项

1. **不要直接修改全局变量**：只使用提供的函数接口
2. **事件通信**：模块间只通过事件通信，不要直接调用其他模块的函数
3. **模块独立**：经济系统必须能独立运行测试
4. **命名规范**：严格遵循规则手册中的变量命名
